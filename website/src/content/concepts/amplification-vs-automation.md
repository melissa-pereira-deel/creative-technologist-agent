There are two fundamentally different things a tool can do. It can do something *for* you — automating a task, removing the need for you to engage with it. Or it can do something *with* you — amplifying your capacity to engage with it more effectively.

A calculator automates arithmetic. After using one, you're the same person — perhaps faster, but no deeper. A spreadsheet amplifies reasoning. After using one seriously, you think differently about relationships between variables. You've built mental models that persist beyond the tool.

Doug Engelbart called this distinction the difference between automating and augmenting. He spent his career arguing that computers were being designed to do things for humans when the deeper opportunity was to extend what humans could do. His 1968 demonstration — the Mother of All Demos — wasn't a productivity showcase. It was a proof of concept for augmented human intellect.

The distinction matters most in design because the choice usually isn't labeled. Most features can be designed to either automate (here's the answer) or amplify (here's how to think about the question). The automating version is often easier to build and easier to use in the short run. The amplifying version is harder, but it leaves the user with something permanent.

## Where it shows up

- **AI features:** does this feature give users answers or help them understand the domain?
- **Dashboard design:** does this show the result or reveal the reasoning behind it?
- **Onboarding:** does this do the first task for the user or teach them to do it?
- **Recommendation systems:** does this replace user judgment or inform it?
